HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta element contains metadata such as character style.
  • URLs are added using the a element for anchor.
  • CSS is added using an external link with 'rel="stylesheet" href="./relative-path-to-css-sheet"'.
  • Classes can be defined to group elements for later group formatting.

CSS

selector {property: value}

  • Selector '*' applies properties to all elements on page.
  • Multiple selectors can have the same rule e.g. 'header, footer {rule}'.
  • Period preceding selector e.g. '.card' applys rules to all elements of class 'card'.
  • Property 'margin' defines the space around the outside of an element.
  • Property 'padding' adds space around the content onside an element.
  • Property 'display' affects image layout; value 'block' makes each image start on a new line.
  • Property 'color' can be given as a named colour or hexadecimal value.

Git

  • git status: checks what branch we are currently on.
  • git pull origin branch-name: syncs changes in server.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A adds all files in working branch.
  • git commit -m "message": adds changes to github.
  • git push origin branch-name: pushes commited changes to server.

JavaScript

  • Powers tools such as Application Programming Interfaces (APIs) that add functionality to a project.
  • Links to HTML using the 'script' element, just before closing 'body' element.
  • Variables are declared with let e.g. let myVariable;
  • Variable types include strings, numbers, bools, arrays ([]), or just general objects.
  • To select certain objects, as we did with CSS classes, we use 'querySelector("selector-value")', e.g. 'document.querySelector("h1")' to select all h1 elements.
  • Comments are enclosed with /* */ or prefaced by //.
  • Equality tests use ===. Equal/not equal are ! or !==.
  • Built in functions include alert(), multiply(), and querySelector().
  • Event handlers such as 'click' add interactivity by running code in response to an event. The function to run is passed to the function addEventListener e.g. 'document.querySelector("html").addEventListener("click", function () { alert("Ouch! Stop poking me!"); });' runs this alert after a click.
  • Unnamed functions such as the one above cana lso be assigned using => e.g. 'document.querySelector("html").addEventListener("click", () => { alert("Ouch! Stop poking me!"); });'
